# Сторож: синхронизация документации с кодом

## Eve-Arch в коде

При любом изменении, где фигурирует маркер **`Eve-arch: <ARTICLE>`** (или добавляется новый артикул):

1. Обнови строку в **`EVE_ARCH_INDEX.md`** (описание, файлы, дата; колонка **Docs-link** должна указывать на актуальный якорь).
2. Обнови **целевой раздел** по ссылке из **Docs-link** — обычно **`ARCHITECTURE.md#…`**, для части паттернов — **`FEATURE_*_CHECKLIST.md#…`** (например синхронизация списка с URL → чек-лист клиентов).
3. Пример: **`FIXED-LAYOUT-001`** → [`ARCHITECTURE.md`](./ARCHITECTURE.md)`#fixed-layout` (stable layout, min-height, таблицы, модалки).

Если правка **не задаёт повторяемый паттерн**, в коде оставляй **`Eve-arch: 000`** и не раздуй индекс; при необходимости одной строкой отрази нюанс в **`DIAGNOSTIC_CHECKLIST.md`** или чек-листе фичи.

## Глобальные формулировки («по всему сайту»)

Сначала **`grep Eve-arch`** по репозиторию, затем согласованно обнови код, индекс и раздел в **`ARCHITECTURE.md`** / чек-листах — см. порядок работы в **`EVE_ARCH_INDEX.md`**.

## Футер в документах

В конце каждого `.md` под `docs/` — блок **«Синхронизация»** со ссылкой на этот файл и на **`EVE_ARCH_INDEX.md`** (относительные пути от каталога файла).
