# Eve Indexes — оглавление

Расширенные реестри помимо **Eve-Arch** (структурные паттерны кода: `Eve-arch:` в `EVE_ARCH_INDEX.md`): бизнес-потоки, UX, производительность, безопасность.

## Индексы

| Индекс | Файл | Сфера |
|--------|------|--------|
| **Eve-BL** | [`EVE_BL_INDEX.md`](./EVE_BL_INDEX.md) | Бизнес-потоки, фазы, сущности ↔ экраны (**индекс заполнен** по текущим фичам + домен) |
| **Eve-UX** | [`EVE_UX_INDEX.md`](./EVE_UX_INDEX.md) | Интерфейс, анимации, обратная связь, доступность взаимодействия |
| **Eve-PERF** | [`EVE_PERF_INDEX.md`](./EVE_PERF_INDEX.md) | Производительность (бандл, рендер, списки, сеть) — **есть артикул** lazy-маршрутов |
| **Eve-SEC** | [`EVE_SEC_INDEX.md`](./EVE_SEC_INDEX.md) | Безопасность (данные, API, XSS, секреты) — **есть артикул** env/apiBaseUrl |

Структурные паттерны layout/API/роутинга по-прежнему в **[`EVE_ARCH_INDEX.md`](./EVE_ARCH_INDEX.md)** (`Eve-arch:`).

**Backend (Express/Prisma):** **[`EVE_BACKEND_INDEX.md`](./EVE_BACKEND_INDEX.md)** — маркеры **`Eve-BE:`** (`API`, `SEC`, `LOG`).

## Порядок работы (фикс / новая фича)

1. **Поиск по проекту** (одним проходом или отдельно):
   - `grep -E "Eve-arch:|Eve-BL:|Eve-UX:|Eve-PERF:|Eve-SEC:"`  
   - или в IDE: `Eve-arch`, `Eve-BL`, `Eve-UX`, `Eve-PERF`, `Eve-SEC`.
2. **Подходит существующий артикул** — используй тот же подход и файлы; в коде оставь/дополни маркер соответствующего семейства.
3. **Не подходит** — заведи новый артикул в **нужном** индексе (таблица: Артикул, Название, Описание, Файлы, Docs-link), затем маркер в коде:
   - **TS/SCSS:** `// Eve-UX: UX-ANIM-001 — кратко` (подставь семейство и код).
   - **HTML:** `<!-- Eve-UX: UX-ANIM-001 — кратко -->`
4. Согласованность: если изменение одновременно структурное и UX — допустимы **оба** маркера (`Eve-arch` + `Eve-UX`) рядом.
5. Перед закрытием задачи проверь универсальный DoD и протокол конфликтов в [`DOCS_SYNC_RULES.md`](./DOCS_SYNC_RULES.md) — "частичное внедрение" не допускается.

**Бизнес-логика (CRUD, статусы, отправка, скидки, цепочки сущностей):** сначала **`grep Eve-BL`**, затем используй существующий артикул из **`EVE_BL_INDEX.md`** или создай новый, **обнови таблицу** и **добавь комментарий в код** — см. раздел «Правило» в [`EVE_BL_INDEX.md`](./EVE_BL_INDEX.md).
